# 医疗设备管理系统说明文档

## 1. 系统概述

医疗设备管理系统是一款以医院医疗设备管理为中心，实现医疗设备全生命周期管理信息化、数字化、智能化为目标的软硬件相结合的医疗设备资产管理系统。系统基于 PHP + MySQL 的技术开发，全部源码开放。

## 2. 系统架构

### 2.1 技术架构

- **后端框架**：ThinkPHP
- **前端框架**：Layui
- **数据库**：MySQL
- **开发语言**：PHP (5.6+)
- **其他技术**：
  - 微信/飞书集成
  - Vue.js (微信版本)

### 2.2 系统模块

系统主要包含以下核心模块：

#### 2.2.1 设备管理模块

- 设备查询
- 设备借调申请
- 设备转科申请
- 设备报废申请
- 设备档案管理
- 设备生命周期管理

#### 2.2.2 维修管理模块

- 设备报修
- 维修工单管理
- 维修记录查询
- 维修统计分析

#### 2.2.3 巡查保养模块

- 巡查计划管理
- 保养计划管理
- 巡查记录
- 保养记录

#### 2.2.4 质控管理模块

- 质控计划管理
- 质控记录管理
- 质控统计分析

#### 2.2.5 采购管理模块

- 采购计划
- 采购申请
- 采购审批
- 采购验收

#### 2.2.6 系统管理模块

- 用户管理
- 角色权限管理
- 部门管理
- 系统配置
- 日志管理

### 2.3 系统特点

1. **全生命周期管理**：覆盖医疗设备从采购、验收、使用、维修、保养到报废的全过程管理。

2. **多端支持**：

   - Web 管理端
   - 移动端
   - 微信公众号
   - 企业微信
   - 飞书（可选）

3. **灵活配置**：

   - 支持分院管理
   - 可配置审批流程
   - 可定制报表和统计分析

4. **数据安全**：

   - 完善的权限管理
   - 操作日志记录
   - 数据备份恢复

5. **扩展性**：
   - 模块化设计
   - 支持二次开发
   - API 接口支持

## 3. 功能详情

### 3.1 设备管理

- **设备档案**：记录设备基本信息、技术参数、附件资料等
- **设备分类**：支持多级分类管理
- **设备编码**：支持自定义编码规则
- **设备状态**：在用、停用、报废、维修中等状态管理
- **设备调拨**：支持设备借调、转科等流程
- **设备报废**：支持报废申请、审批流程

### 3.2 维修管理

- **报修申请**：支持在线报修
- **维修派工**：支持维修任务分配
- **维修处理**：记录维修过程、更换配件等信息
- **维修验收**：维修完成后的验收流程
- **维修统计**：设备故障率、维修及时率等统计分析

### 3.3 巡查保养

- **计划制定**：支持周期性巡查保养计划制定
- **任务分配**：支持巡查保养任务分配
- **执行记录**：记录巡查保养执行情况
- **异常处理**：巡查保养中发现的异常处理流程

### 3.4 质控管理

- **质控计划**：支持设备质量控制计划制定
- **质控执行**：记录质控执行情况
- **质控分析**：质控数据统计分析

### 3.5 采购管理

- **需求收集**：各部门设备需求收集
- **采购计划**：制定采购计划
- **采购申请**：发起采购申请
- **采购审批**：多级审批流程
- **采购验收**：设备到货验收流程

### 3.6 系统管理

- **用户管理**：用户添加、编辑、权限分配
- **角色管理**：角色定义、权限设置
- **部门管理**：部门结构设置
- **系统参数**：系统基础参数配置
- **日志管理**：系统操作日志查询

## 4. 系统接口

### 4.1 微信集成

- 微信公众号登录

- 微信扫码功能

### 4.2 飞书集成（可选）

## 5. 系统要求

### 5.1 服务器要求

- **操作系统**：Windows/Linux
- **Web 服务器**：Apache/Nginx
- **PHP 版本**：5.6.40 及以上
- **MySQL 版本**：5.5 及以上
- **存储空间**：建议 10GB 以上

### 5.2 客户端要求

- **PC 端**：支持主流浏览器（Chrome、Firefox、Edge 等）
- **移动端**：支持 Android、iOS 系统
